DirectX 11 3D Game Demo

Textured & lit level • data-driven loading • animated character • camera & skybox

C++ DirectX 11 HLSL Animation Data-driven

At a Glance

  • Built a small first-person level with textured & lit meshes, alpha-tested foliage, and an HDRI skybox.
  • Implemented data-driven level loading from a simple level.txt format (meshes, lighting, skybox, camera).
  • Animated T-Rex driven by a lightweight state machine (Idle → Run → Attack) reacting to player distance.
  • Responsive camera: WASD movement, mouse-look, clamped pitch, and a subtle walking “bounce”.

Overview

The demo showcases a compact DirectX 11 pipeline: opaque and alpha-tested meshes rendered with per-pixel lighting, an animated character, and a skybox that tracks the camera to seal the horizon. The scene targets two moods—night and day—by swapping skylight intensity and ambient terms in the light buffer.

Lighting combines ambient, diffuse and specular components in the pixel shader (half-vector specular, shininess 32). This keeps materials legible while avoiding overblown highlights in night scenes.

Content is loaded from a human-readable level file: meshes, animation type, skybox texture/radius, and lighting vectors are declared as key-value pairs, parsed at start-up to construct the world without recompilation.

Key Systems

1) Data-Driven Level Loading

A minimal format (level.txt) declares objects such as T-Rex, Skybox, and Lighting. Vectors are parsed from comma-separated triples, and properties are mapped onto components (mesh path, initial position, HDRI, skylight parameters).

2) Rendering & Materials

Opaque meshes receive standard ambient+diffuse lighting. Alpha-tested meshes (e.g., foliage) discard transparent fragments in the pixel shader to keep crisp silhouettes.

3) Character Animation & Interaction

The T-Rex uses a simple controller to switch between Idle, Run and Attack as the player approaches. Skinning is performed in the vertex shader by blending up to four bone matrices per vertex.

4) Camera & Feel

First-person controls combine WASD translation with mouse-look. Pitch is clamped, the cursor recenters every frame, and a sine-based “walking bounce” adds subtle locomotion feedback.

5) Skybox

Code Highlights

Pixel Lighting (ambient + diffuse + specular)
// In PS: half-vector specular (shininess 32) with skylight intensity
float3 L  = normalize(LightDirection);
float3 N  = normalize(input.Normal);
float3 V  = normalize(-input.Pos.xyz);
float  dif = max(dot(L, N), 0.0);
float3 H  = normalize(L + V);
float  spe = pow(max(dot(N, H), 0.0), 32.0);
float3 color = AmbientColour
             + SkylightColour * (dif * LightIntensity)
             + SkylightColour * (spe * LightIntensity * 0.5);
Alpha Testing for Foliage
// Keep crisp leaves/grass edges
if (color.a < 0.5f) discard;
Skinned Vertex Transform (4-bone blend)
// Weighted sum of bone matrices in VS
float4x4 T = bones[input.BoneIDs[0]] * input.BoneWeights[0];
T += bones[input.BoneIDs[1]] * input.BoneWeights[1];
T += bones[input.BoneIDs[2]] * input.BoneWeights[2];
T += bones[input.BoneIDs[3]] * input.BoneWeights[3];
output.Pos = mul(input.Pos, T);

Evaluation & Outlook

The level integrates rendering, animation, input, and data-driven content into a cohesive playable slice. Future extensions include instancing for larger scenes, deferred shading, and adding shadow/normal mapping for greater depth.
