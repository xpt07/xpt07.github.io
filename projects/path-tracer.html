Project

Interactive Path Tracer

Realistic CPU Rendering with BVH, MIS & Denoising

C++ Path Tracing DirectX Multithreading OIDN BVH MIS

Overview

A physically-based renderer written in C++, implementing Monte Carlo path tracing with support for area lights, emissive materials, complex BSDFs, multiple importance sampling (MIS), and Open Image Denoise (OIDN) for final-frame filtering. The engine is modular and supports scene loading via a custom binary format.

Highlights

  • BVH acceleration structure with SAH-inspired layout
  • Multiple importance sampling for efficient lighting
  • Layered BSDF system supporting rough conductors and dielectrics
  • Importance sampling of HDR environment maps via precomputed CDFs
  • Tile-based multithreaded rendering and per-tile adaptive sampling
  • Intel OIDN denoiser with albedo + normal auxiliaries

Architecture

The system follows a modular architecture:

  • Core: Vectors, rays, intersection tests, transforms
  • Geometry: Triangles, AABBs, BVH
  • Materials: BSDF hierarchy with diffuse, mirror, conductor, coating layers
  • Lights: Area lights, emissive triangles, HDR environment
  • Integrator: Recursive path tracing with direct lighting and MIS
  • Film: HDR accumulation, filters, tonemapping, denoising

Performance

On a modern 6-core CPU, renders of 1920×1080 resolution with 8192 spp complete in under 10 minutes using multithreaded tile dispatch. Adaptive sampling showed limited gains in typical scenes.

Challenges & Lessons

  • Balancing MIS weights to avoid bias when mixing delta and diffuse paths
  • Stabilizing the OIDN denoiser with coherent normal/albedo buffers
  • Debugging layered BSDF path interactions (e.g. coating + mirror)

How to Run

  1. Clone the repository from GitHub (link below)
  2. Open the Visual Studio solution and build in Release mode
  3. Run with command line args: -scene materials -spp 8192
  4. Use WASD + EQ for camera; press P to save the image
