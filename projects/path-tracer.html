Interactive Path Tracer

Realistic CPU Rendering with BVH, MIS & Denoising

C++ Path Tracing DirectX Multithreading OIDN BVH MIS

How It Works

This project implements a CPU-based path tracer using BVH, MIS, OIDN, and layered BSDFs in a modular architecture. It delivers high-quality global illumination rendering while offering real-time interaction capabilities.

1. BVH Acceleration

The BVH accelerates ray-triangle intersection. Below is a visualisation of the spatial hierarchy traversal alongside source code from Geometry.h.

if (ray.intersects(node.bounds)) {
    if (node.isLeaf()) {
        checkTriangles(node);
    } else {
        traverse(node.left);
        traverse(node.right);
    }
}

2. BSDF Layering

The tracer supports multiple material models, including rough dielectric, conductor, and Fresnel blend layers. The diagram below shows a stack of BSDFs applied on an object’s surface.

3. Importance Sampling (MIS)

For complex lighting environments, MIS combines BSDF and light sampling. The tabulated 2D distribution improves HDR environment convergence.

5. Light Tracing

Light tracing paths are launched from lights to improve caustics. Connection attempts to the camera are weighted using sensor response.

// Sensor weight in light tracing

How to Run

  1. Clone: https://github.com/xpt07/RTBase.git
  2. Open in Visual Studio (Release mode)
  3. Run: -scene materials -spp 8192
  4. Controls: WASD + EQ, P to save image
