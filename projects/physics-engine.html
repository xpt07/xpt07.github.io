Project

2D Custom Physics Engine

C++ rigid-body dynamics with collision detection and an OpenGL + ImGui sandbox

C++17 Rigid Bodies Impulse Solver OpenGL GLFW/GLAD Dear ImGui Premake

Overview

A lightweight 2D physics engine written from first principles in C++. It supports rigid-body dynamics and narrow-phase collision detection for Circle–Circle, Circle–OBB, and OBB–OBB pairs, with impulse-based resolution (restitution + friction) and positional correction. A real-time OpenGL sandbox—driven by Dear ImGui—provides scene management, live object editing, and on-the-fly creation of new bodies.

Summary

  • Clear architecture: Engine core is decoupled from rendering and UI (scenes + physics vs. sandbox + ImGui). RAII (`std::unique_ptr`) and singletons used judiciously for lifecycle control.
  • Mathematically correct contact handling: Collision normals, relative velocities, clamped friction impulses, and Baumgarte-style positional correction, implemented and explained.
  • Practical tooling: Per-scene GUI editors to select bodies, edit materials/pose, and spawn new circles/OBBs at runtime—useful for both debugging and demonstrations.
  • Production-style build + docs: Premake→Visual Studio generation, precompiled headers, and Doxygen entrypoint prepared for API docs.

Architecture at a glance

  • Core: Application (main loop), OpenGLWindow (GLFW/GLAD), Gui host, and a custom vector2.
  • Physics: RigidBody with forces/impulses, Material (mass, restitution, friction), and shapes Circle, OBB.
  • Collisions: CollisionUtility implements circle-circle, circle-OBB (closest-point test), and OBB–OBB overlap using centers/extents; resolution applies normal/tangential impulses and positional correction.
  • Scenes: Scene (mixed), CircleScene, OBBScene, each with its own onGui() for object selection, editing, and creation.
  • Sandbox: OpenGL renderer + Dear ImGui; left sidebar switches scenes; right panel shows the active scene’s editor.

Technical highlights

  • Impulse solver: Computes normal impulse from the relative velocity projected on the contact normal; restitution applied conditionally; tangential friction impulse clamped by μ |J_n|.
  • Positional correction: Penetration depth corrected with a small slop and a configurable percentage to avoid sinking and jitter.
  • Per-scene editors: Each scene exposes onGui() to: (i) select an existing body; (ii) adjust position, rotation, mass, restitution, friction, and static/dynamic type; (iii) spawn new objects.
  • Rendering path: Immediate-mode OpenGL for clarity; GLFW for windowing; GLAD for function loading.

Code highlight — impulse with friction (simplified)

// Given contact normal n, relative velocity vrel, inverse masses invA, invB
float vn = dot(vrel, n);
float jn = -(1.f + restitution) * vn / (invA + invB);   // normal impulse
vec2  impulseN = jn * n;

// Apply normal impulse
A.velocity -= invA * impulseN;
B.velocity += invB * impulseN;

// Tangential direction
vec2 t = vrel - dot(vrel, n) * n;
if (length(t) > 1e-6f) t /= length(t);

// Friction impulse, clamped by μ|jn|
float jt = -dot(vrel, t) / (invA + invB);
float jt_clamped = clamp(jt, -mu * fabs(jn), mu * fabs(jn));
vec2 impulseT = jt_clamped * t;

// Apply friction impulse
A.velocity -= invA * impulseT;
B.velocity += invB * impulseT;

Try it in the sandbox

  • Switch to Circle Scene and drag positions; toggle Static and adjust restitution to observe bounciness changes on impact.
  • Switch to OBB Scene, rotate a box and add a new OBB; watch normal vs. friction impulses affect sliding vs. sticking.
  • Use the mixed Scene to test Circle–OBB interactions and positional correction (no interpenetration drift).
