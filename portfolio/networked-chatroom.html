Project

Networked Chatroom

Multi-threaded client/server chat • C++ • Winsock • ImGui

C++ Winsock Multithreading ImGui DirectX

Overview

A full-stack C++ chat application demonstrating a robust client–server architecture. The server manages multiple clients concurrently over Winsock; the client provides a modern GUI via ImGui/DirectX, supporting broadcast and private messaging.

Key Features

  • Multi-client server with thread-per-connection and safe synchronization.
  • Broadcast and private messages using a simple @username: convention.
  • Live user list broadcast from the server.
  • Responsive ImGui client with real-time message view and input.

Technical Notes

  • Networking: Winsock, non-blocking sockets; clean session handling.
  • Concurrency: std::thread, mutex/atomics; graceful shutdown.
  • GUI: ImGui with DirectX rendering; separate IO and render loops.
  • Protocol: @mentions for DMs; USERLIST: server pushes.

Code Highlights

// Broadcast a message to all clients except the sender
void broadcast_message(const std::string& msg, SOCKET sender) {
    std::lock_guard lock(clients_mtx);
    for (auto& [sock, user] : clients) {
        if (sock != sender) send_line(sock, msg);
    }
}

Challenges & Lessons

  • Avoiding race conditions across reader/writer threads.
  • Designing a minimal but expressive message protocol.
  • Balancing real-time rendering with network IO on the client.
