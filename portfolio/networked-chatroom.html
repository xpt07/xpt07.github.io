Overview

This Networked Chatroom project is a full-stack C++ application developed to demonstrate real-time communication over a client-server architecture. The project is split into two parts:

  • Server Side: Handles multiple client connections using the Winsock API, routes both broadcast and direct (private) messages, manages connection states, and maintains an up-to-date list of connected users.
  • Client Side: Features an interactive graphical user interface (GUI) built with ImGui, integrated with DirectX, to display messages, manage user input, and initiate both group and private conversations.

Key Features

  • Robust Client-Server Architecture using Winsock for network communication.
  • Support for Broadcast Messaging as well as Private Messaging using an �@username:� syntax.
  • Concurrent handling of multiple clients with multi-threading and synchronization (mutex, atomic variables).
  • Dynamic User List: The server continuously broadcasts an updated list of active users.
  • Interactive GUI Client developed using ImGui for a responsive, modern interface.

Challenges & Lessons Learned

  • Ensuring thread-safety while managing concurrent client connections and avoiding race conditions.
  • Parsing and handling both broadcast and private messages effectively using a simple command protocol.

Demo & Instructions

To run the project:

  1. Compile the server code (WinSockReverseServer.cpp) using your preferred C++ compiler on Windows.
  2. Run the server executable. It will listen for connections on port 65432.
  3. Compile the client code (WinSockReverseClient.cpp) and run the executable. The client GUI will launch and connect to the server.
  4. Use the prompt to enter a username and start sending messages. Click on the usernames on the left to open private windows and send private messages.
